MySQL은 오픈소스 관계형 데이터베이스입니다.

관계형 DB란 데이터를 테이블 구조로 저장하고,
테이블 간 관계를 외래키로 표현하며,
SQL로 데이터를 조작하는 방식입니다.

MySQL의 기본 스토리지 엔진은 InnoDB인데,
트랜잭션, MVCC 기반 동시성 제어, 외래키 제약을 지원합니다.

실무에서 MySQL을 많이 선택하는 이유는
오픈소스라 비용 부담이 없고,
커뮤니티와 레퍼런스가 방대하며,
AWS RDS, Aurora 같은 클라우드 환경과 통합이 잘 되어 있기 때문입니다.

PostgreSQL과 비교하면,
MySQL은 읽기 성능과 복제 구조에 강점이 있고,
PostgreSQL은 복잡한 쿼리, JSON, 확장 타입 지원이 더 강합니다.
대용량 트래픽 읽기 위주 서비스엔 MySQL,
복잡한 데이터 모델엔 PostgreSQL을 많이 선택합니다.

MySQL 같은 관계형 DB는 스키마가 고정되어 있고
트랜잭션과 정합성이 강점입니다.
MongoDB 같은 NoSQL은 스키마가 유연하고
수평 확장이 쉽지만 트랜잭션이 약합니다.
정합성이 중요한 주문, 결제엔 MySQL,
로그, 피드처럼 구조가 유동적인 데이터엔 NoSQL을 씁니다.
