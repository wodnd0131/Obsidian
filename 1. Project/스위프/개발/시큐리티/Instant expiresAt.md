

`Instant`는 자바 8부터 도입된 새로운 날짜/시간 API(`java.time` 패키지)의 핵심 클래스로, **"타임스탬프(Timestamp)"를 표현하는 타입**입니다.

정확히는 UTC(협정 세계시) 기준 1970년 1월 1일 0시 0분 0초(에포크 타임)를 기준으로 현재까지 시간이 얼마나 흘렀는지를 나노초(ns) 단위의 정밀도로 나타냅니다.

스프링 부트 3.x 환경이나 현대적인 자바 백엔드 개발에서 토큰 만료 시간(`expiresAt`), 생성일(`createdAt`), 수정일(`updatedAt`) 등을 데이터베이스와 연동할 때 가장 권장되는 타입입니다.

### 1. `Instant`의 핵심 특징

- **기준은 오직 UTC (타임존 없음):** 한국 시간(KST), 미국 시간(EST) 같은 특정 지역의 시차가 반영되어 있지 않습니다. 오직 영국 그린위치 천문대 기준인 UTC+0 시각만을 가집니다.
    
- **불변 객체(Immutable):** `Instant` 객체의 값을 한 번 생성하면 변경할 수 없습니다. 값을 더하거나 빼면(`plus()`, `minus()`) 기존 객체가 변하는 게 아니라 새로운 `Instant` 객체가 반환되므로 멀티스레드 환경에서 안전합니다.
    
- **나노초 단위 정밀도:** 밀리초(ms)를 넘어 나노초($10^{-9}$초) 단위까지 아주 정밀하게 시간을 기록합니다.
    

### 2. 왜 다른 타입(`LocalDateTime`, `Date`) 대신 `Instant`를 쓰나요?

자바에는 시간을 나타내는 여러 타입이 있습니다. 어떤 차이가 있는지 비교해 보면 `Instant`를 쓰는 이유를 쉽게 알 수 있습니다.

| **타입**              | **타임존(시차)**     | **설명**                                                     | **주요 용도**                            |
| ------------------- | --------------- | ---------------------------------------------------------- | ------------------------------------ |
| **`Instant`**       | **없음 (UTC 고정)** | 타임존의 영향을 받지 않는 절대적인 타임스탬프                                  | **로그인 토큰 만료**, 서버 로그 기록, DB 생성/수정 시간 |
| `LocalDateTime`     | 없음 (서버 로컬)      | 시차 개념이 없는 "말 그대로의 숫자" (예: 2026년 12월 25일 13시)               | 알람 예약, 타임존이 필요 없는 순수 날짜/시간           |
| `ZonedDateTime`     | **있음 (지역 포함)**  | 특정 지역의 시차가 포함된 시간 (예: 2026-12-25 13:00 +09:00[Asia/Seoul]) | 회원에게 거주 국가 기준으로 시간을 보여줄 때            |
| `Date` / `Calendar` | 과거의 유산          | 자바 8 이전에 쓰던 타입으로, 버그가 많고 멀티스레드에 취약함                        | **사용 금지 (레거시)**                      |

> **💡 왜 `expiresAt`에 `Instant`가 딱일까요?**
> 
> 서비스 서버가 한국에 있든, 미국 AWS 리전에 있든, 전 세계 어디서나 똑같은 기준점(UTC)으로 토큰 만료를 계산해야 시간 오류가 생기지 않기 때문입니다.

### 3. 실제 코드로 보는 활용법

Java

```
// 1. 현재 이 순간의 UTC 시간 가져오기
Instant now = Instant.now(); 
System.out.println(now); // 출력 예시: 2026-06-10T06:41:00Z (끝에 'Z'가 UTC를 뜻함)

// 2. 만료 시간 계산 (예: 현재 시간으로부터 1시간 뒤 만료)
Instant expiresAt = now.plus(Duration.ofHours(1));

// 3. 만료 여부 확인하기 (현재 시간이 만료 시간을 지났는지?)
if (Instant.now().isAfter(expiresAt)) {
    System.out.println("토큰이 만료되었습니다.");
}
```

### 4. 만약 한국 시간(KST)으로 변환해서 보고 싶다면?

`Instant`는 내부적으로 UTC로만 돌기 때문에, 개발자가 로그를 보거나 화면에 뿌려줄 때 한국 시간으로 바꾸고 싶다면 아래와 같이 타임존을 붙여서 변환하면 됩니다.

Java

```
Instant instant = Instant.now();

// UTC 기준 Instant를 한국 시간(Asia/Seoul)으로 변환
ZonedDateTime kstTime = instant.atZone(ZoneId.of("Asia/Seoul"));
System.out.println(kstTime); // 출력: 2026-06-10T15:41:00+09:00[Asia/Seoul]
```
