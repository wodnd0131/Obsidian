## 전체 인증 플로우

```
[로그인]
클라이언트 POST /auth/login
  → SecurityConfig: permitAll 이라 필터 통과
  → LoginController: @Valid 로 입력값 검증
  → LoginServiceImpl.login(): DB 조회 → BCrypt 비밀번호 비교
  → JwtTokenProvider.generateToken(): access + refresh 토큰 각각 발급 (HS256)
  → DB upsert: refresh 토큰의 jti를 계정당 1개 레코드로 저장
  → 응답: { access_token, refresh_token }

[인증이 필요한 API 호출]
클라이언트 GET /any-api (Authorization: Bearer <access_token>)
  → JwtAuthenticationFilter: 헤더에서 토큰 추출 → 서명·만료 검증
  → 검증 성공: SecurityContext 에 Authentication 저장
  → 컨트롤러 진입 허용

[토큰 재발급]
클라이언트 POST /auth/refresh (body: refresh_token)
  → LoginServiceImpl.refreshTokens(): refresh 토큰 서명·만료 검증
  → DB 의 jti 와 비교 → 탈취된 토큰(이미 재발급된 토큰) 감지
  → 새 access + refresh 토큰 발급 (Refresh Token Rotation)
  → DB 의 jti 갱신
```
