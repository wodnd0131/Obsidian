
가장 필요한 것은 관심과 노력이라고 생각합니다.
해당 직군이 뭘 필요로하는지, 또 내가 필요한 것을 무엇인지 소통하는 것이 출발점이라 생각합니다.
하지만, 이렇게 관심과 노력을 쏟는 것은 시간이 많이드는 작업입니다.
그래서 저는이 시간 비용을 최적화에 집중 했습니다.

먼저 매일 데일리 미팅으로 작업내용을 공유하며, 문제 토론 환경을 유지했습니다.
예로, 도메인 구조를 수정되며 프론트 작업까지 지연되는 문제를 발견할 수 있었습니다.
그리고 이를 해결하기 위해 API 버저닝을 진행하였구요.

인터셉터를 활용하여 스웨거 상 표시되는 API 의 버전 별 정렬과 레거시 표시 추가를 진행했습니다.
