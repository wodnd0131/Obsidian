```
import java.util.*;
class Solution {
    public boolean solution(String[] phone_book) {
        Arrays.sort(phone_book);
        for (int i = 1; i < phone_book.length; i++) {
        if(phone_book[i].startsWith(phone_book[i-1])){
                return false;
            }
        }
        return true;
    }
```


## 왜 "바로 앞 원소"만 봐도 충분한가

`A`가 `B`의 접두사라고 해봐요 (`A = "119"`, `B = "1199"`). 정렬 기준으로 `A < B`인 건 자명하죠. 근데 혹시 그 사이에 다른 문자열 `C`가 끼어들 수 있지 않을까? 즉 `A < C < B`인 `C`가 있어서 둘이 떨어질 수 있지 않나?

답은 "아니오"예요. `C`가 `A`보다 사전순으로 크고 `B`보다 작으려면, `A`의 길이만큼은 `C`도 `A`와 똑같아야 해요. `A`의 글자 중 하나라도 다르면 그 지점에서 이미 `A`보다 크거나 작은 게 확정돼버려서 `A < C < B` 사이에 들어올 수가 없거든요 (`B`가 `A + "9"`인데, `A`의 어느 자리에서 벗어난 `C`는 `B` 범위 안에 들어올 수 없음). 즉 `C`도 결국 `A`를 접두사로 가질 수밖에 없어요.

**그래서 정렬해두면, 어떤 문자열이 다른 문자열의 접두사라면 반드시 바로 인접한 자리에 온다**는 게 보장돼요. 전체 쌍을 다 비교(O(n²))할 필요 없이 인접한 것만 봐도 되는 이유가 이거예요.

## 시간복잡도 짚기

- `Arrays.sort()`: O(n log n)
- 반복문: O(n) × `startsWith()` 비용

여기서 하나 주의할 점 — `startsWith()`은 O(1)이 아니라 **O(m)**이에요 (m = 비교하는 문자열 길이). 문자열 비교는 글자 하나하나 대조해야 하니까요. 그래서 정확히는 O(n log n × m) 정도인데, 전화번호 길이가 제한적이라 실전에서는 O(n log n)으로 취급해도 무방해요.

## 참고로 해시로도 풀 수 있는 문제예요

`Set<String>`에 전화번호부 다 넣어두고, 각 번호의 모든 접두사(`"119"`라면 `"1"`, `"11"`, `"119"`)를 하나씩 잘라서 Set에 있는지 확인하는 방식도 가능해요. 근데 이건 접두사 자르는 과정 자체가 O(m)이라 오히려 지금 정렬 방식이 더 깔끔하고 빠릅니다. **"정렬해서 인접만 비교"가 이 문제의 정석 풀이**라고 보시면 돼요.
