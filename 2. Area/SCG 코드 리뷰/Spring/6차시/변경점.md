

### 1. 의존성 추가

- `spring-boot-starter-validation` 추가

### 2. 커스텀 예외 4종 신설

|예외|용도|
|---|---|
|`ReservationNotFoundException`|존재하지 않는 예약 조회/삭제|
|`DuplicateReservationException`|동일 사람/시간 중복 예약|
|`ReservationFullException`|시간대 정원 초과|
|`InvalidReservationTimeException`|영업시간 외 예약|

### 3. Controller 계층 검증

- `ReservationRequest` record에 `@NotBlank`, `@NotNull` 어노테이션 추가
- `createReservation()` 파라미터에 `@Valid` 적용
- 반환 타입을 `Reservation` 도메인 객체 → `ReservationResponse` DTO로 전환

### 4. GlobalExceptionHandler 확장

- 커스텀 예외 4개 각각 핸들러 추가
- `MethodArgumentNotValidException` 핸들러 추가

### 5. Domain 불변화

- `Reservation` 필드 전부 `final`로 변경
- setter 전체 제거
- 기본 생성자를 `protected`로 제한
- 내부 검증 예외를 `IllegalArgumentException` → `InvalidReservationTimeException`으로 교체

### 6. Repository 개선

- `orElse(null)` → `orElseThrow(ReservationNotFoundException::new)`
- 삭제 로직: `removeIf` 방식 → 먼저 조회 후 없으면 예외, 있으면 삭제

### 7. Service 정리

- `IllegalArgumentException` → 의미에 맞는 커스텀 예외로 교체
- `Reservation` 도메인 객체 직접 반환 → `ReservationResponse` DTO 변환 후 반환

### 8. 테스트 추가

- 필수 필드 누락 시 400, 없는 예약 삭제 시 400 응답 검증

---

한 마디로: **형식 검증(Controller), 비즈니스 규칙(Service), 불변식(Domain)** 세 레이어를 각각 정비하고, `IllegalArgumentException` 하나로 퉁쳤던 예외를 의미 단위로 분리한 커밋들이야. 이제 리뷰 들어갈까?
