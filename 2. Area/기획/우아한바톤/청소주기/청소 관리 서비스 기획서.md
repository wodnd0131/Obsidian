## 1. 서비스 한 줄 정의

**청소를 해야 하는 상태를 보여주고, 그걸 탭 몇 번으로 직접 해결하거나 전문 서비스로 맡길 수 있게 연결해주는 생활 위생 관리 플랫폼.**

```
청소 주기 관리 + 문제 해결 가이드 + 용품·서비스 연결 + 구독 자동화
```

더 긴 설명: 사용자는 집안 청소를 항목별로 기억하지 않아도 된다. 앱이 마지막 완료일을 기준으로 상태(정상/임박/지연)를 자동 계산해 보여주고, 지연된 항목을 누르면 직접 청소하는 방법, 필요한 용품, 맡길 수 있는 서비스까지 한 화면의 흐름 안에서 확인하고 실행할 수 있다.

## 2. 배경

- **마이크로 외주 서비스의 성장**: 세탁 수거, 쓰레기 배출, 소형 청소 단위의 서비스 등장 및 확산
- **생활 관리 도구의 수요 증대**: 청소 주기 관리 앱(HomeTasker, Sweepy, Tody)의 검증된 사용자 수요

### 2.2 기존 서비스의 한계

|서비스 유형|제공하는 것|부족한 것|사용자 불만|
|---|---|---|---|
|청소 트래커 앱|주기 관리, 체크리스트|해결 수단|밀림을 보여주고 끝난다|
|가사청소 업체|해결 수단|발견 시점|사용자가 먼저 찾아와야 한다|
|세탁·수거·청소 플랫폼|특정 서비스|통합 비교|매번 검색하고 비교하기 번거롭다|
|포털 검색|정보의 양|신뢰와 맥락|매번 처음부터 검색, 정보가 쌓이지 않는다|

### 2.3 핵심 인사이트

```text
❌ 사용자는 청소를 싫어하는 것이 아니다.

   도구를 준비하는 과정이 너무 번거롭기 때문에 미룬다.

❌ 청소 서비스 가격이 비싼 것이 아니다.
✅ 단건 결제의 심리적 부담이 있고,
   필요한 시점을 모르고, 여러 플랫폼을 비교하는 수고가 크다.
```

## 4. 문제 정의

### 4.1 페인포인트 1: 인식과 실행의 실패 — "청소는 눈에 보이지 않으면 의지도 생기지 않는다"

**문제의 본질**

집안일은 항목마다 주기가 제각각이라 머릿속으로 관리하는 것이 불가능하다.

- 빨래, 이불: 1~2주
- 욕실 물때: 2~4주
- 세탁조 청소: 1~2개월
- 에어컨 필터: 2~4주
- 곰팡이 관리: 3~4주

화장실에 곰팡이가 슬고 냄새가 나는 '위생의 임계점'을 넘어야만 심각성을 깨닫고, 미뤄둔 청소는 손대기 싫은 '중노동'이 되어버린다.

인식했다고 해도 실행이 곧바로 이어지지 않는다. 주말에 큰맘 먹고 시작하려는 순간 곰팡이 제거제가 떨어져 있거나, 어떤 세제를 써야 할지 몰라 흐름이 끊기고 포기하게 된다. **인식의 실패와 실행의 마찰이 겹쳐 방치의 악순환이 만들어진다.**

**솔루션**

청소를 각 잡고 하는 노동에서 일상적 '자기 관리'로 전환한다.

- 직관적인 색상 변화(초록 ➜ 노랑 ➜ 분홍)로 주기를 한눈에 파악
- 지연된 항목만 탭 가능하게 만들어 불필요한 상호작용 제거
- 주기가 임박한 시점에 필요한 용품을 선제적으로 추천해 실행의 마찰 제거
- 한계에 다다랐을 때 **자동으로 전문 서비스를 제안**하여 방치 방지

### 4.2 페인포인트 2: 파편화된 해결 수단 — "빨래, 쓰레기, 청소... 앱을 몇 개나 깔아야 해?"

**문제의 본질**

1인 가구는 필요에 따라 다양한 앱을 깔아야 한다:

- 세탁: 런드리고, 세탁특공대, 크린토피아
- 쓰레기 수거: 오늘수거, 커버링
- 가사청소: 미소, 청연
- 청소용품: 쿠팡, 네이버쇼핑, B마트

각 앱마다 가격, 조건, 지역 가능성이 다르다. 그리고 서비스를 받기로 결정해도 **어떤 업체가 좋은지, 평판이 어떤지 비교할 정보가 없다.** 매번 검색하고 여러 플랫폼을 오가며 비교하는 수고가 너무 커서, 결국 외주화마저 포기하게 된다.

**솔루션**

모든 가사 O2O 서비스의 **통합 게이트웨이(Gateway)**가 된다.

- 대시보드 하나에서 내 주기를 확인
- 각 분야 검증된 파트너 업체 3~5개를 큐레이션해 가격·평점·후기·서비스 범위를 한눈에 비교
- 사용자 후기와 실제 매칭 기록 기반의 신뢰도 표시, 파트너사는 명시적 제휴 배지로 표시
- 버튼 한 번에 예약까지 연결

## 5. 포지셔닝

### 5.1 경쟁 환경 분석

|경쟁사|핵심 기능|장점|한계|
|---|---|---|---|
|**HomeTasker/Sweepy/Tody** (청소 트래커)|주기 관리, 체크리스트|사용자가 항목을 자유롭게 커스텀|선입력 비용이 높다 / 해결 수단이 없다 / 게임화에만 집중|
|**런드리고/크린토피아** (세탁 전문)|세탁 수거 예약|특화된 서비스 품질|세탁만 가능 / 주기 관리 없음 / 매번 찾아서 호출해야 함|
|**오늘수거** (수거 전문)|쓰레기 수거|저가 단위 / 반복 가능|수거만 가능 / 다른 청소 수요와 통합 안 됨|
|**미소/청연** (가사청소 전문)|청소 서비스 예약|프로페셔널 팀|초기 예약 수고 / 정기 관리 어려움 / 주기 자동화 없음|

### 5.2 차별점 1: 선입력 비용을 완전히 제거

**기존 트래커의 한계**: 사용자가 항목을 등록해야 작동 → **클린루프**: 가입 시점을 '집이 마지막으로 정돈된 날'로 간주해 자동 계산

```text
기존: 항목 등록 → 주기 설정 → 알림 대기
클린루프: 가입 즉시 → 상태 자동 계산 → 대시보드에 즉시 노출
```

### 5.3 차별점 2: 상태에서 해결까지 한 화면 안에서 완성

기존 서비스들은 **하나만** 잘한다. 트래커는 상태를 보여주고 끝나고(주기 관리만), 업체 플랫폼은 사용자가 먼저 찾아와야 하고(서비스만), 커머스는 용품만 판다.

→ **클린루프**: 상태 확인 → 가이드 → 도구·용품 → 서비스 비교 → 자동 주기 등록을 한 화면에서 완성

```text
한 화면의 4단계 플로우:
① 주제 확인 (지연된 항목만 탭 가능)
② 세부 문제 선택 (예: 욕실 곰팡이 → 타일 곰팡이)
③ 해결 가이드 (단계별 방법, 소요시간, 주의사항)
④ 도구·업체 비교 (다나와식 가격비교 + 업체 카드)
→ 해결 완료 시 확인 절차 없이 자동으로 다음 주기 등록
```

직접 할지 외주할지의 판단, 필요한 세정제 구매, 신뢰할 수 있는 업체 예약이 모두 이 플로우 안에서 끝난다. **청소판 다나와에 주기 관리가 결합된 형태.**

### 5.4 차별점 3: 수동적 리마인드보다 능동적 상태 표시

**기존 방식**: 알림에만 의존 → **클린루프**: 홈 진입 시 **대시보드**의 청소 주기 카드로 상태를 한눈에 노출

- 정상(초록 체크): 탭 불가능 → 불필요한 상호작용 없음
- 임박(노랑 점): 알림 채널로만 리마인드
- 지연(분홍 점선): 직접 탭 가능 → 능동적 행동 유도

**입장이 바뀌면 UI도 달라진다**: "아직 할 일이 없네"에서 "지금 해야 할 것이 명확하네"로 변환

## 6. 핵심지표

### 6.1 North Star Metric

```
해결된 지연/임박 항목 수 (월간)
— 해결 방식: SELF(직접) / PARTNER(업체 예약) 분리 추적
```

성공 기준: 직접 완료와 업체 예약이 고르게 분포

- SELF만 많음 → 거래가 없는 유틸리티에 머물렀다는 뜻
- PARTNER 있음 → 중개 수수료 기반 수익화 검증됨

### 6.2 생사를 가르는 지표

|지표|정의|검증 가설|
|---|---|---|
|**푸시 권한 획득률**|(알림 허용) / (온보딩 완료)|알림은 이 서비스의 핵심 재방문·수익 채널이다. 권한을 못 얻으면 모델의 절반이 작동하지 않는다|
|**대시보드 진입률**|(지연 카드 탭 수) / (지연 카드 노출 수)|상태 표시가 실제로 행동을 유도하는가?|
|**4단계 플로우 완료율**|(해결 완료 또는 예약까지 도달) / (플로우 시작)|사용자가 중간에 이탈하지 않고 끝까지 진행하는가?|
|**업체 예약 전환율**|(예약 버튼 클릭) / (업체 비교 화면 노출)|정보 비교에서 실제 거래로 전환되는가?|
|**주간 클리어 달성률**|(그 주의 지연·임박 카드를 모두 처리한 사용자) / (WAU)|주 단위 완료 루프가 재방문 습관을 만드는가?|
|**D7 / D30 리텐션**|가입 후 7일/30일 재방문율|첫 알림이 도착하기 전에 이탈하지 않는가? (페르소나의 '3일 만에 삭제' 극복 여부)|

## 7. MVP

### 7.1 MVP 핵심 범위

**만드는 것** (최소한의 완성도로 가설 검증)

|#|기능|설명|
|---|---|---|
|1|**온보딩**|폼 없음. 소셜 로그인만으로 가입. 가입 시점 = 상태 계산 기준선. 알림의 가치를 먼저 설명하는 **사전 설득 화면 후 푸시 권한 요청** (권한 획득률이 생사 지표)|
|2|**대시보드**|6개 카테고리의 청소 주기 카드 상태 표시 (정상/임박/지연). 임박·지연을 한눈에 확인하고, 완료 시 주기가 갱신되는 홈 페이지. 지연 항목만 탭 가능|
|3|**해결 플로우 4단계**|주제 확인 → 세부 이슈 선택 → 가이드 → 도구·업체 비교|
|4|**도구 가격비교**|다나와식 최저가 표시 (정적 큐레이션 데이터)|
|5|**업체 비교 카드**|이름/가격/후기/평점/CTA, 제휴 배지 명시, 예약 가능|
|6|**해결 완료 자동 등록**|확인 절차 없음. 완료 즉시 대시보드 카드·주기 반영|
|7|**주간 클리어 배지**|그 주의 지연·임박 카드를 모두 처리하면 "이번 주 클리어!" 배지. 주 단위 완료 루프의 보상 장치|
|8|**주기 보드**|완료 항목 리스트, 마지막 완료일, 다음 주기 표시|
|9|**알림(벨)**|지연 항목 능동 리마인드, 업체 비교 화면으로 딥링크|
|10|**하단 탭바**|홈 · 청소(가운데 강조) · 주기 보드|

**만들지 않는 것** (백로그로 이동)

|제외|이유|
|---|---|
|업체 예약 결제 연동|예약 UI는 시연하되 결제 시스템은 이후|
|구독/크레딧 모델|중기 확장 계획으로 이동|
|업체 실시간 API 연동|초기는 정적 큐레이션 데이터로 충분|
|사진 기반 AI 진단|이후 단계. 초기는 입력 기반 상태 계산|
|커뮤니티 (후기, 팁)|백로그. MVP는 핵심 해결 루프만|
|대화형(채팅) UI|프로토타입 제작 중 기각|
|검색 (벡터 매칭)|MVP 범위 초과. 상태 기반 진입만으로도 충분|

### 7.2 리텐션 설계: 주간 클리어 루프

청소 항목의 주기는 길게는 1~2개월이다. 긴 주기 항목만 있으면 사용자는 첫 알림이 오기 전에 앱을 삭제한다 — 페르소나 김보송이 체크리스트 앱을 3일 만에 지운 이유다. 클린루프는 이를 **주기 길이가 다른 항목의 조합**으로 해결한다.

```text
짧은 주기 항목(빨래·이불 1~2주, 쓰레기 배출 주 단위)
  → 매주 최소 1개 이상의 항목이 '임박/지연' 상태로 전환
  → 매주 앱을 열 이유가 자연스럽게 생성
  → 모든 카드를 처리하면 주간 클리어 배지가 완료의 보상으로 작동
  → 짧은 주기 항목으로 형성된 방문 습관이
     긴 주기 항목(세탁조, 에어컨 필터)의 처리율까지 견인
```

즉 리텐션을 별도의 게임화 장치로 만들지 않고, **주기 데이터 자체가 재방문 사이클을 만들도록** 설계한다. 이 루프가 작동하는지는 주간 클리어 달성률과 D7/D30 리텐션 지표로 검증한다.

### 7.3 프로토타입에서 반드시 보여줄 2가지 시나리오

#### **시나리오 A: 대시보드에서 시작하는 직접 해결**

```
장면 1. 김보송이 앱을 열면 대시보드의 주기 카드들이 보인다.
       "욕실·곰팡이 11일 지연" (분홍 점선)

장면 2. 대시보드의 욕실 카드를 탭
       → 하단 시트 올라옴, STEP 1 (주제 확인)

장면 3. "타일 곰팡이" 카드 선택
       → STEP 3 (가이드: 원인, 난이도, 단계별 방법)

장면 4. [청소 도구·업체 보기]
       → STEP 4 (도구 가격비교, 업체 카드)

장면 5. [해결 완료] 탭
       → 등록 토스트 메시지
       → 대시보드 카드가 즉시 지연→완료(초록)로 전환
       → 시트 닫히며 주기 보드로 화면 전환
```

**보여주는 가치**: 상태 확인 → 가이드 → 용품/서비스 → 자동 주기 갱신이 한 화면에서 완성됨

#### **시나리오 B: 알림에서 시작하는 외주화**

```
장면 1. 홈 우측 상단 벨에 안 읽음 점
       탭하면 "🗑️ 쓰레기 배출이 지연되고 있어요"

장면 2. 알림 탭
       → 수거 서비스 비교 시트
       → 업체 3개 카드:
         1️⃣ 오늘수거: 주 2회, 15,900원/월 ⭐ 가장 많이 선택
         2️⃣ 커버링: 주 1회, 9,900원/월
         3️⃣ 로컬 수거팀: 맞춤형 견적

장면 3. [오늘수거로 예약] 탭
       → 예약 완료 토스트
       → 쓰레기·수거 카드가 "처리됨" 상태로 전환
       → 다음 주기 자동 갱신 (주 2회 기준)

장면 4. 두 항목이 모두 처리되어 대시보드의 모든 카드가 정상 상태
       "이번 주 클리어!" 배지가 표시된 홈 화면
       → 주간 클리어 루프(7.2)의 보상 순간
```

**보여주는 가치**: 알림을 통해 지연된 항목의 신뢰할 수 있는 업체를 빠르게 비교하고 예약할 수 있음

## 8. 확장 계획

### 8.1 단기 (0~3개월): 초기 진입과 MVP 검증

**초기 진입 전략 (Go-to-Market)**

청소 관리는 저관여 카테고리라 "필요할 때" 검색해서 깔리는 앱이 아니다. 따라서 **생활 위생을 새로 설계해야 하는 순간**을 진입 모멘트로 삼는다.

|요소|전략|
|---|---|
|**진입 모멘트**|자취 시작·이사 직후. "집이 정돈된 날 = 가입일" 가정이 실제로 참이 되는 유일한 시점이라, 제품 로직과 획득 타이밍이 일치한다|
|**채널 가설**|자취·원룸 커뮤니티와 대학 커뮤니티(에브리타임 등)에 청소 꿀팁·주기 콘텐츠로 유입. 광고보다 콘텐츠 기반 오가닉을 우선해 초기 CAC를 최소화|
|**시즌 집중**|이사·개강 시즌(2~3월, 8~9월)에 획득 리소스 집중|
|**회수 논리**|초기엔 저CAC 오가닉 중심으로 확보하고, 사용자당 중개 수수료 기반 LTV를 측정해 유료 획득 확장 여부를 판단|

**검증할 것**

- 푸시 권한 획득률이 충분히 높은가? (온보딩 사전 설득 화면의 효과)
- 대시보드 진입률이 충분히 높은가?
- 4단계 플로우 완료율이 50% 이상인가?
- 알림 → 파트너 랜딩 전환율이 대시보드 직접 탭과 비슷한가?
- 주간 클리어 루프가 D7/D30 리텐션으로 이어지는가?

**추가 기능**

- 알림 설정 화면 (빈도 조절, 항목별 on/off, calm mode)
- 검색 재도입 (벡터 매칭 또는 간단한 텍스트 검색)
- 계절성 카테고리 추가 (장마철 곰팡이, 겨울 결로 등)

### 8.2 중기 (3~9개월): 개인화, 공동 사용, 구독 모델

- **개인화**: 사용자 행동 기록 기반 추천, 문제별/용품별 후기 축적, 커뮤니티 라운지
- **공동 사용**: 가구 멤버 초대와 담당 배정 (가족, 룸메이트)
- **구독·크레딧 모델**: 월 생활 위생 예산처럼 크레딧을 충전해 단건 결제의 심리적 부담을 완화하고, 정기 수거·청소를 구독으로 자동 갱신. 반복 거래 데이터는 파트너 제휴가 협상의 지렛대가 된다

### 8.3 장기 (9개월~): B2B 확장 및 플랫폼화

주기 관리 + 업체 비교라는 코어는 청결이 곧 매출인 B2B 타겟(개인 숙박시설 운영자, 원룸 임대관리자, 쉐어하우스 등)으로 확장 가능하다. 플랫폼 측면에서는 데이터 기반 선제 파트너 매칭 고도화, 업체 대시보드, 결제·정산 자동화, 사진 기반 AI 상태 진단으로 발전시킨다.

## 9. 비즈니스 모델 (MVP 단계)

### 9.1 운영 모델: 애그리게이터

|모델|초기 청소부 고용|운영 리스크|확장 속도|
|---|---|---|---|
|기존 O2O 앱|필수 (직접 팀 구성)|높음 (급여, 교육, 이탈)|느림|
|**클린루프**|불필요|낮음|빠름|

클린루프는 **'주기 데이터'만 쥐고 모든 전문 파트너사를 입점**시킨다. 마치 배달의민족이 요리사를 고용하지 않는 것처럼, 우리는 청소부를 고용하지 않는다.

### 9.2 초기 수익원 (2가지)

|수익원|구조|화면 구현|시점|
|---|---|---|---|
|**중개 수수료**|업체 예약 시 거래 수수료 (CPA/CPS)|업체 비교 카드의 예약 CTA|사용자가 예약 완료 시|
|**커머스 제휴**|용품 구매 링크 클릭 시 수수료|도구 가격비교 카드|사용자가 구매 완료 시|

**핵심**: MVP 단계에서는 두 가지 거래로 충분히 검증 가능

### 9.3 수익 논리

**1단계**: 주기 데이터 축적 → 사용자의 행동 패턴으로 "언제 어떤 서비스가 필요한지" 파악

**2단계**: 알림 채널의 가치 (가설) → 지금 정확히 지연 중인 항목의 사용자만 타겟팅 가능 → 파트너 입장에서 "의도가 명확한 수요"이므로 일반 배너 광고 대비 높은 전환율을 낼 것으로 가정하며, **이 가설은 알림 → 예약 전환율 지표로 검증**한다

**3단계**: 확장 (중기) → 구독 데이터로 파트너와 제휴가 협상 → 반복 수요를 예측하고 공동구매 가능

### 9.4 과금 원칙 (초기)

**모든 사용자에게 무료 제공**

- 청소 주기 관리
- 문제 해결 가이드
- 업체·용품 비교 및 예약

**향후 (중기)에 추가할 유료 항목**

- 가족/공동 사용 (멤버 동기화)
- 구독 크레딧 (월 생활 위생 예산)
- 고급 반복 설정 (N주 단위, 계절별)
- 기록 분석 및 통계
- 광고 제거

**전략**: MVP는 "정보 + 비교 + 거래 수수료"로 시작, 확실한 수요가 있으면 구독 추가

### 11.1 설계 변경 이력 (실제 제작 중 내린 결정들)

|해결 완료 시 확인 chip|확인 절차 없이 자동 등록|해결했다는 사실 자체가 등록 신호|

### 11.2 정직성 원칙

- **업체·용품·요금제 데이터는 큐레이션된 정적 데이터**임을 발표에서 먼저 밝힌다
- **제휴 관계가 있는 항목에는 명시적으로 '제휴' 배지**를 단다
- 사용자 후기와 실제 완료 기록이 축적되면 이를 다음 추천에 반영한다
