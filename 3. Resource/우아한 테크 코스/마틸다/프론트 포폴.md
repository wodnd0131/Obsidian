# TilGuy FE — 황재웅 작업 포트폴리오  
  
> 프로젝트: TIL(Today I Learned) 기록 서비스 프론트엔드  
> 기간: 2025.03 ~ 2025.07  
> 역할: 프론트엔드 개발 (개발환경 구축 · 컴포넌트 설계 · CI/CD)  
  
---  
  
## 1. 개발 환경 구축  
  
### 배경  
팀 초기에 로컬 환경 불일치로 빌드 재현성이 보장되지 않았고, 배포 파이프라인이 수동 관리되어 통합 오류 감지가 늦었다.  
  
### 수행 내용  
  
#### 프로젝트 초기 세팅 (`f437a46`, `89ea6c8`, `7e7539c`)  
- Vite + React 기반 초기 구조 설계  
- ESLint 규칙 및 의존성 명세 통일 → 팀원 간 코드 스타일 편차 제거  
  
#### GitHub Actions CI/CD 파이프라인 구축 (`f96c806`, `9e4495d`, `17a85f3`, `3915f54`, `1519253`)  
- `gitaction build` 워크플로우 작성 및 빌드 자동화  
- 빌드 권한 설정 · EOF 지정 오류 · 불필요한 헬스체크 순차 제거로 파이프라인 안정화  
  
#### 공통 HTTP 클라이언트 구성 (`3746f9e`, `50cf7e9`)  
- `AxiosInstance` 단일 진입점으로 캡슐화 (baseURL, 인터셉터 일원화)  
- 이후 모든 API 호출이 해당 인스턴스를 통해 이루어지도록 강제 → 인증 토큰 주입 및 에러 처리의 공통화 기반 마련  
  
#### 인증 라우팅 환경 (`8ff18bb`, `96182df`, `c5b0d49`)  
- `ProtectedRoute` 컴포넌트 구현으로 인증 여부에 따른 접근 제어 분리  
- 새로고침 시 인증 상태 유실 문제 수정 및 권한 만료 시 자동 로그아웃 처리

## 2. OOP 관점의 컴포넌트 · 스타일 관리  
  
### 배경  
개발 초기 컴포넌트 내부에 UI 로직, 비즈니스 로직, 스타일이 혼재하여 재사용이 어렵고 변경 영향 범위가 커지는 문제가 있었다.  
  
### 핵심 사고  
> "단일 책임 원칙(SRP)을 기준으로, 컴포넌트는 '무엇을 보여주는가'만 알고 — '어떻게 동작하는가'는 util/hook으로 분리한다."  
  
### 수행 내용  
  
#### 공통 유틸리티 모듈화 (`97304b1`, `9ad5035`, `323272a`)  
- `DateUtils` 도입: 날짜 포맷 로직을 UI에서 완전 분리  
- 파일명 camelCase 통일 (`DateUtils.js → dateUtils.js`, `AxiosInstance.js → axiosInstance.js`) → OS별 대소문자 인식 차이로 인한 빌드 오류 사전 차단  
  
#### 컴포넌트 추상화 및 분리 (`f9ec90e`, `df1feea`, `f6abbfc`, `5df6222`, `c1d17e2`)  
- `ActionButton` 공통 컴포넌트 도출: 북마크·삭제·수정 등 반복되던 버튼 UI를 단일 인터페이스로 통일  
- `AlltilsPage` 분리: 목록 렌더링과 페이지 레이아웃 로직을 독립된 컴포넌트로 추출  
- 포스트 생성 페이지 컴포넌트 분리: 한 파일에 집중된 폼·미리보기·날짜 선택 로직을 각 관심사에 맞게 분리  
  
#### 스타일·구조 일관성 확보 (`2d3d66a`, `22e4771`, `718483a`, `157249a`)  
- 인라인 스타일 → 별도 스타일 파일 분리로 UI 계층과 스타일 계층 명확히 구분  
- sticky footer 전역 적용으로 레이아웃 정책 일원화  
- 패키지 재배치 및 컨벤션 통일로 모듈 간 결합도 관리  
  
#### 주석 처리 안정화 (`3140363`, `3aea034`)  
- 특수문자·중복·연속 주석이 마크다운 파싱 오류를 유발하는 엣지케이스 대응  
- 파싱 전처리 로직 추가로 사용자 입력 허용 범위 확장
